# FAQ

Frequently asked questions for contributors to `@v14/ui`.

---

## General

**Q: Where do I start?**

Read [Getting Started](../getting-started/README.md), then run `pnpm storybook` to explore existing components before writing anything.

**Q: I found a bug. How do I report it?**

Open an issue on GitHub. Use the Bug Report template and include a minimal reproduction.

**Q: I want to add a new component. Where do I begin?**

Open a Feature Request issue to align on the design before writing code. Check whether the component exists in Ark UI — if it does, wrap it rather than reimplement it.

## PandaCSS & Styled System

**Q: What is `src/styled-system/` and can I edit it?**

It is auto-generated by `panda codegen`. Never edit it manually — changes will be overwritten on the next codegen run. Edit `panda.config.mjs` instead, then re-run `pnpm prepare`.

**Q: When do I use `css()` vs `cva()` vs `sva()`?**

- `css()` — one-off inline styles, not reusable
- `cva()` — component with visual variants (single root element)
- `sva()` — component with variants across multiple slots (e.g. a Card with header, body, footer)

For components, always prefer `sva()` — it keeps variant logic co-located with the slots.

**Q: How do I add a new design token?**

Add it to `panda.config.mjs` under `theme.extend.tokens` or `theme.extend.semanticTokens`, then run `pnpm prepare`.

## TypeScript

**Q: Why is `exactOptionalPropertyTypes` enabled?**

It catches the difference between `prop: undefined` and omitting `prop` entirely — a frequent source of silent bugs in component APIs. Do not work around it with `??: undefined` casts; fix the type instead.

**Q: The build fails with type errors I didn't introduce.**

Pull the latest `main` and run `pnpm install` — a dependency or the generated styled-system may have changed.

**Q: How do I type a component that forwards a ref?**

```ts
import { forwardRef } from "react";

export interface ButtonProps extends React.ComponentPropsWithoutRef<"button"> {
  intent?: "primary" | "secondary";
}

export const Button = forwardRef<HTMLButtonElement, ButtonProps>(
  ({ intent = "primary", ...props }, ref) => {
    return <button ref={ref} {...props} />;
  }
);
Button.displayName = "Button";
```

## Testing

**Q: Where do tests live?**

Next to the component file: `src/components/Button/Button.test.tsx`.

**Q: How do I run a single test file?**

```bash
pnpm vitest run src/components/Button/Button.test.tsx
```

**Q: Do I need to test every component?**

Yes. Every exported component must have at minimum: a render smoke test, keyboard interaction coverage, and an accessibility check with `@testing-library/jest-dom`.

## Storybook

**Q: My story doesn't show styles.**

Make sure `src/global.css` is imported in `.storybook/preview.ts` and that PandaCSS has been run (`pnpm prepare`).

**Q: What file extension do stories use?**

`.stories.tsx` only. Files named `.stories.ts` or `.stories.js` are not picked up by the Storybook glob.

## Contributing

**Q: How large should a PR be?**

A PR should be reviewable in under 30 minutes. One component per PR is the right scope.

**Q: Do I need to write tests for every change?**

Yes for new components and bug fixes. Documentation-only and story-only PRs are exempt.

**Q: Who merges PRs?**

Maintainers merge after one approving review and all CI checks pass (check-types, tests, build).
